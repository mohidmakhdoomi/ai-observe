KEY_ISSUES:
- `LiveTracer.start()` is planned to open `.trace` and `.jsonl` synchronously, but the plan does not say what `run()` should do if one of those opens fails before the thread is started. Given the spec’s fallback guarantee, this should be called out explicitly as “warn and continue with post-hoc-only behavior,” not left implicit.
- The new failure branches repeatedly say “strict-mode applies,” but they do not explicitly preserve the current wrapper contract of reporting the original Codex exit on stderr before returning `1` in strict mode. This is especially underspecified for the **join-timeout** branch and the **live-parser error → fallback** branch.
- `test_live_parse_disabled_matches_post_hoc_only` is too weak if it only compares final `.jsonl` contents, because live mode should also converge to the same final file. The plan should require an explicit assertion that `LiveTracer` was never constructed/started when `CODEV_OBSERVE_LIVE_PARSE=0`.
