The spec handles this (read returns empty → sleep poll interval), but it's worth confirming the thread doesn't raise on an empty initial read. The spec's approach (just sleep and retry) is correct.

In practice, the thread reads a local file and strace has exited, so this is unlikely — but a `join(timeout=...)` with a fallback would be defensive. Non-blocking.
