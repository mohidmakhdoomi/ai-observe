### Code Quality Issues

**1. Minor: `copy_file_range` doesn't check `writable_fds` (Low severity)**

`splice` requires `fd not in state.writable_fds` → return None, but `copy_file_range` and `sendfile` only check that the fd/path is known, not that the fd is writable. This is a deliberate design choice since `copy_file_range` writes to the output fd regardless of open flags, and `sendfile` similarly writes to its `out_fd`. Both syscalls will succeed even if the fd was only opened for writing (and wouldn't have been opened otherwise). The spec says "when destination fd/path can be resolved safely" — and the implementation correctly requires `path is not None` as the safety check. This is fine.

**2. Minor: `sendfile` argument index (Correct but worth noting)**

`sendfile(out_fd, in_fd, offset, count)` — the implementation uses `args[0]` as the destination fd, which is correct per the Linux `sendfile(2)` signature where the first argument is the output fd.

**3. Good: `copy_file_range` skips unknown fds safely**

When fd is unknown (args[2] = 99 with no path annotation), the test `test_copy_file_range_and_sendfile_modify_known_destination` line 5 shows it's silently skipped — safe false negative as required.

**4. Good: Meta sidecar design is solid**

The `build_session_meta` function cleanly maps the three session modes (normal, timeout-rebuilt, parser-failure) to artifact roles, and the test verifies the rebuilt timeout path end-to-end.
