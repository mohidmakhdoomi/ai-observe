### What Phase 5 Requires But Is Missing

Comparing against the plan's Phase 5 success criteria and the spec's Viewer provenance UX MUST requirements:

**1. No provenance rendering in the UI (MUST — Spec & Plan)**

> "Render event provenance clearly enough that users can distinguish `strace/direct` from `snapshot/inferred`."

- **`table.js`**: No source badges, no provenance column, no visual indicator. Rows show Path/Bytes/Events/Last Touched only.
- **`treemap.js`**: `tooltipFor()` (line 78) shows `path + Bytes + Events + Last touched` — no source/confidence.
- **`index.html`**: No UI elements for provenance display at all.
- **`style.css`**: No badge/provenance CSS classes.

**2. No source filtering (MUST — Spec & Plan)**

> "Add source filtering or equivalent visibility controls for at least `strace` and `snapshot` events."

- **`index.html`/`index.js`**: No source toggle buttons/checkboxes (like the existing "Show filtered" checkbox but for strace/snapshot sources).
- **`aggregator.js`/`_aggregator_oracle.py`**: Neither propagates `source`/`confidence` through aggregation entries, nor implements source-based filtering semantics. Events arrive with provenance in SSE payloads, but the aggregator ignores `source` completely.

**3. No source/confidence propagation in aggregation (Plan)**

> "propagate source/confidence through aggregation, implement source filtering semantics alongside existing path filters, and track source composition per node where useful"

- `newEntry()` in aggregator.js has no source tracking
- `_PathEntry.__slots__` in _aggregator_oracle.py has no source tracking
- Snapshot tree nodes don't carry source composition

**4. No artifact banner support (MUST — Spec & Plan)**

> "Sibling `<session>.jsonl.partial`, `<session>.jsonl.rebuilt`, and `<session>.meta.json` are detected and displayed through a non-sensitive banner/control."

- **`server.py`**: No `/meta` endpoint, no sidecar detection logic
- **`index.html`/`index.js`**: No banner DOM element, no meta-state rendering
- **`tailer.py`**: No meta.json reading or artifact precedence logic for the viewer

**5. Missing test fixtures and coverage (Plan)**

> "Add fixtures for mixed v1/v2 strace/snapshot JSONL streams and golden aggregation output."

- No mixed v1/v2 fixture files exist in `tests/fixtures/viewer/`
- No source filter behavior tests
- No badge/tooltip rendering tests
- No server/tailer meta sidecar detection tests
- No artifact banner tests
