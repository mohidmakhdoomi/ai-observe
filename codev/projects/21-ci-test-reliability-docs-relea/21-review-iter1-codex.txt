- `.github/workflows/ci.yml` masks real test failures in both test-running steps. Each step does `set -o pipefail` and then runs `python -m unittest ... | tee ...`, but without `set -e` or an explicit check of the pipeline status, the script continues to the later `grep` and can exit 0 if there were no skips. In bash, `pipefail` alone does not abort the script. Result: a failing suite/smoke run can be reported as green.
