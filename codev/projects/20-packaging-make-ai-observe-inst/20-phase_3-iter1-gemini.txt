### Detailed Feedback

1. **`setuptools` Dependency in `sys.executable`**
   In [tests/test_packaging_smoke.py](file:///home/user/code/ai-observe_root/ai-observe/.builders/spir-20/tests/test_packaging_smoke.py#L83), the smoke test uses the current test interpreter `sys.executable` to run the PEP 517 build backend snippet:
   ```python
   proc = _run([sys.executable, "-c", snippet, srcdir, distdir])
   ```
   Modern Python (3.12+) virtual environments do not ship with `setuptools` by default. If a developer runs `pytest` or `unittest` inside the local `.venv`, `sys.executable` will point to the venv's Python, and the setup block will fail with `ModuleNotFoundError: No module named 'setuptools'`.
   
   *Recommendation*: Modify the build helper to check if `setuptools` is available in `sys.executable` first, and if not, try to fall back to `python3` (which is highly likely to have `setuptools` installed globally on Linux development hosts), or raise a clear assertion error advising the developer to run `pip install setuptools` inside their venv.

2. **Clean up `HTTPError` to Silence `ResourceWarning`**
   In `_ViewerProc.get` in [tests/test_packaging_smoke.py](file:///home/user/code/ai-observe_root/ai-observe/.builders/spir-20/tests/test_packaging_smoke.py#L185-L186):
   ```python
   except urllib.error.HTTPError as exc:
       return exc.code, exc.read()
   ```
   Because `HTTPError` behaves as a file-like response object and holds onto the socket connection, leaving it unclosed causes Python to issue:
   `ResourceWarning: Implicitly cleaning up <HTTPError 404: 'Not Found'>`
   
   *Recommendation*: Clean up the socket connection by ensuring `exc.close()` is called:
   ```python
   except urllib.error.HTTPError as exc:
       try:
           return exc.code, exc.read()
       finally:
           exc.close()
   ```
