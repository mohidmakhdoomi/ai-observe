## Implementation Review: Phase 1 — Filter Core

### 1. Spec Adherence ✅

**Factory defaults**: The 8 factory filter patterns are exact glob equivalents of the previously hardcoded regex patterns. Golden snapshot tests pass without modification, proving identical behavior.

**Glob syntax**: All spec-mandated examples pass:
- `/tmp/**` → matches `/tmp`, `/tmp/a`, `/tmp/a/b` ✓
- `/home/*/.cache/**` → matches `/home/alice/.cache`, `/home/bob/.cache/pip/x` ✓
- `/work/build/*` → matches `/work/build/a.o`, not `/work/build/obj/a.o` ✓
- `/work/build/**` → matches `/work/build`, descendants ✓
- `/work/?.txt` → matches `/work/a.txt`, not `/work/ab.txt` ✓
- Regex metacharacters escaped (`/work/a+b.[txt]`) ✓

**Validation**: Non-absolute patterns rejected. Empty/whitespace-only rejected. Clear error messages.

**Event-level rule**: All-non-empty-paths-match rule applied correctly with custom filters.

**Exact vs subtree**: `/work/build` does not hide `/work/build/out.log`; `/work/build/**` does. ✓

**Tombstones**: Tombstoned paths hidden even when `include_noise=True`. ✓

### 2. Code Quality ✅

- Clean separation: glob compilation → filter matcher → aggregator consumption. Each layer is independently testable.
- Backward-compatible API: `isNoise`/`eventIsNoise` remain as aliases, `NOISE_PATTERNS` alias points to factory patterns. Existing callers won't break.
- `createAggregator(opts)` accepts both `filter_patterns` (snake_case) and `filterPatterns` (camelCase), which is pragmatic for JS/Python interop.
- No `eval`, `innerHTML`, or `document.write` in the aggregator — security requirement met.
- `FACTORY_FILTER_PATTERNS` is frozen in JS and a tuple in Python — immutable.
- `snapshot()` accepts both `include_noise` (backward compat) and `include_filtered` (new name) — clean migration path for phase 3.

### 5. Minor Observations (non-blocking)

- `glob_to_regex_source` is public (no underscore) in the Python oracle but module-private in JS. This is fine since the oracle is a test module, not a public package.
- A pattern like `/a//b` (double slash) compiles without error, producing a regex matching the literal string `/a//b`. This is harmless — unusual input, predictable output.
- The `compileSegmentGlob` function treats `*` inside a non-`**` segment as segment-local, which means a pattern like `/foo/a**b/bar` treats the stars as two `[^/]*` rather than a subpath wildcard. This is correct per spec but might surprise users. Not a phase-1 concern — validation could warn on suspicious patterns in a later phase.
