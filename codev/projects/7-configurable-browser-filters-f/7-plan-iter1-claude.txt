### 5. Risk Assessment

The plan identifies the right risks. One additional consideration:

**Non-standard `/tmp/**` matching `/tmp` itself.** The spec explicitly requires `/tmp/**` to match both `/tmp` and `/tmp/a/b`. Standard glob semantics would NOT match the parent path itself. The Phase 1 success criteria correctly state this requirement, but the glob compiler implementation needs to handle this edge case deliberately. A builder using a standard glob library or implementing naïve `**` expansion could miss this. The criteria are specific enough that a careful builder will catch it, but it's worth noting as a subtle implementation trap.
